How AR Navigation Works
Open Mobile App
Launch the SVES AR companion app on your smartphone. The app connects to the venue IoT mesh via BLE beacon triangulation.
Point at Any Pod
The camera activates AR overlays showing pod ID, capacity, remaining battery, and estimated departure time in real-time.
Follow AR Path
Turn-by-turn AR arrows guide you through the venue to your assigned pod boarding station, avoiding congested corridors.
Board & Go
Scan your booking QR on arrival. The pod door opens automatically. AI-managed levitation takes you to your destination.

Implementation Roadmap
Phase 1 — Foundation
Weeks 1–4
  • Cloud Run + Firestore provisioning
  • IoT sensor mesh hardware deployment
  • MQTT pod telemetry protocol
  • Auth service + API gateway
Phase 2 — Intelligence Layer
Weeks 5–8
  • Gemini API routing integration
  • Queue optimization microservice
  • BigQuery streaming analytics
  • Digital twin simulation
Phase 3 — Attendee Experience
Weeks 9–12
  • Flutter AR mobile app build
  • Pod booking + slot engine
  • Push notification dispatch
  • Staff HUD + emergency protocols
Phase 4 — Hardening
Weeks 13–16
  • Load testing: 80K concurrent users
  • Circuit breakers + failover
  • Edge deployment <100ms latency
  • Staff training + venue integration
Phase 5 — Launch
Weeks 17–20
  • Pilot event deployment (10K cap)
  • Feedback loops into AI models
  • CI/CD pipeline + monitoring
  • Full 80K capacity rollout
Cloud Run Microservices — Specs
pod-manager
Fleet state machine, MQTT telemetry ingestion, emergency halt broadcast, levitation field control.
2 vCPU · 512MB · min 2 instances
queue-optimizer
Dijkstra + ML-based crowd load balancing. Real-time rerouting triggered by density thresholds.
4 vCPU · 1GB · autoscale 2–20

Gemini 2.0 Flash integration for optimal route computation, crowd predictions, anomaly detection.
2 vCPU · 1GB · min 1 instance
notification
Push, SMS, WebSocket dispatch. In-venue LCD screen refresh. AR overlay triggers via BLE beacon mesh.

analytics-ingestion
